## Documentation Standards

### File Naming
- Use `UPPERCASE.md` for top-level documents (README, CHANGELOG, etc.)
- Use `PascalCase.md` or descriptive names for subdirectory docs

### Structure
- Start with a clear title and brief description
- Include a table of contents for long documents
- Use consistent heading hierarchy
- Include code examples where helpful

### Maintenance
- Update CHANGELOG.md with every release
- Keep ROADMAP.md current with development progress
- Review and update technical docs when code changes
- Link between related documents
